# 1. Purpose

This document defines the complete user journey of the AI Placement Platform.

Rather than describing individual screens, this document explains how users move through the product from their first visit to achieving placement.

Every future feature should fit naturally into this journey.

---

# 2. Journey Philosophy

The platform is designed around one central idea:

> Every interaction should move the user one step closer to getting placed.

Users should never wonder:

- What should I do next?
- Where should I go?
- Am I improving?

The platform always answers these questions.

---

# 3. Primary User Persona

# 5. Stage 1 — Discovery

Goal

Introduce the platform.

User arrives from:

- Google
- LinkedIn
- College
- Friends
- Social Media

Questions in the user's mind:

What is this?

Can it help me?

Is it trustworthy?

Platform response:

- Professional landing page
- Career Orbit™ preview
- Feature highlights
- Testimonials
- Clear CTA

Outcome

User clicks "Get Started."

# 17. Motivation Strategy

Motivation should come from progress rather than gamification.

Examples

Resume Score Increased

Career Orbit Improved

Interview Confidence Increased

Profile Completed

Skill Learned

Visual progress is more valuable than badges alone.

---

# 18. Friction Reduction

The platform should minimize unnecessary effort.

Strategies

- Auto-save forms
- Remember preferences
- Smart defaults
- Helpful suggestions
- Clear navigation
- Progressive disclosure

---

# 19. Success Indicators

Users should regularly experience:

"I understand my current position."

"I know what to do next."

"I can see my progress."

"I feel closer to getting placed."

---

# 20. Failure Prevention

The platform should prevent users from becoming stuck.

Every screen should include:

Explanation

Next Action

Helpful Guidance

Support if required

---

# 21. Accessibility Journey

The entire journey should remain accessible.

Requirements

Keyboard Navigation

Screen Readers

Responsive Layout

High Contrast

Reduced Motion

Every stage should remain fully usable.

# 24. Governance

Every new feature should answer:

Which stage of the user journey does it improve?

If no clear answer exists, the feature should be reconsidered.
